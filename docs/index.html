DELETE Method - Qtivity API

Qtivity provides an API to support active call release functionality. This document outlines how to use the DELETE method to release active calls on the Qtivity platform gracefully.

By using the Cisco GUID, active calls can be released with ease.

△

Session-Lookup must be configured with Cisco-Guid in the Qtivity Portal. Please find the reference screenshot.

Example Request URL

DELETE endpoint format 
DELETE http://{backend_server_host}:4000/api/calls/Cisco-Guid/{actual_cisco_guid}

Replace {backend_server_host} with your configured Qtivity backend host and {actual_cisco_guid} with the unique Cisco GUID of the call you want to release.

Sample Request URL

DELETE request example 
DELETE http://qtivity-backend.example.com:4000/api/calls/Cisco-Guid/fb84cb3c-e20c-46d1-860e-532bea367c72

This API call triggers the release of an active call from the Qtivity platform.

Success Response (200 OK)

JSON response 
{
  "success": true
}

Error Response (404 Not Found)

JSON response 
{
  "message": "Channel not found"
}

Expected Content Type for API Requests When making API requests to Qtivity, specify the correct content type to ensure successful communication.

Method DELETE
Protocol http
Backend host {backend_server_host}
Port 4000
Path parameter {actual_cisco_guid} identifies the active call to release.
